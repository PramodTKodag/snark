## What is Snark?

Snark is a public REST API that serves AI-generated humor — roasts, excuses, corporate jargon, fortune cookies, and 25+ other endpoints. Every response is uniquely generated by an LLM. No two responses are ever the same.

- **Zero authentication** — fully public, no API keys needed
- **Multi-provider** — supports Groq (free), Gemini, and Claude with automatic fallback
- **Rate limited** — 50 requests/hour per IP
- **Self-hosted** — run it on your own infrastructure

## Quick Start

### Prerequisites

- Python 3.12+
- Docker & Docker Compose
- PostgreSQL and Redis (or use Docker)

### 1. Clone and configure

```bash
git clone https://github.com/PramodTKodag/snark.git
cd snark
cp .env.example .env
```

Edit `.env` and add your API key for at least one provider:

```bash
# Groq is free — get a key at https://console.groq.com/keys
GROQ_API_KEY=your-groq-api-key-here
```

### 2. Run with Docker

```bash
docker compose --profile dev up --build
```

The API will be available at `http://localhost:8100`.

### 3. Try it out

```bash
# Get a creative excuse to say no
curl http://localhost:8100/v1/wit/say-no/

# Get roasted
curl http://localhost:8100/v1/wit/roast/dave/

# Explain quantum physics like you're 5
curl "http://localhost:8100/v1/wit/explain-like-im-5/?q=quantum+physics"

# Generate an honest git commit message
curl http://localhost:8100/v1/wit/commit-message/
```

### Mood Parameter

Add `?mood=` to any endpoint to change the tone:

```
sarcastic, angry, funny, sad, excited, dramatic,
passive-aggressive, philosophical, wholesome, unhinged
```

Example: `curl "http://localhost:8100/v1/wit/motivation/?mood=passive-aggressive"`

## Architecture

```
snark/
├── base/              # Django config
└── wit/
    ├── models.py      # Persona, ResponseLog
    ├── views.py       # API endpoints
    ├── services.py    # Orchestrator (caching, anti-repetition, fallback)
    ├── providers/     # AI provider abstraction
    │   ├── base.py        # AIProvider interface
    │   ├── registry.py    # Provider registry with fallback
    │   ├── groq_provider.py
    │   ├── gemini_provider.py
    │   └── claude_provider.py
    └── management/    # seed_personas command
```

### Provider Fallback

Snark automatically falls back between AI providers. If the primary provider hits a content filter or rate limit, it retries with a softened prompt, then falls back to the next provider. Configure your preferred default in `.env`:

```bash
AI_DEFAULT_PROVIDER=groq  # groq (free), gemini, or claude
```

## Configuration

All configuration is via environment variables. See `.env.example` for the full list.

| Variable | Default | Description |
|----------|---------|-------------|
| `AI_DEFAULT_PROVIDER` | `groq` | AI provider: `groq`, `gemini`, or `claude` |
| `GROQ_API_KEY` | — | Groq API key (free at console.groq.com) |
| `GEMINI_API_KEY` | — | Google Gemini API key (optional) |
| `ANTHROPIC_API_KEY` | — | Anthropic API key (optional, paid) |
| `POSTGRES_DB` | `snark` | PostgreSQL database name |
| `REDIS_DB` | `9` | Redis database number |
